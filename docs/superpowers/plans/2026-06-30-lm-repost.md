# Lead-Magnet Repost — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a one-click "Repost" for an already-published lead magnet that regenerates fresh, fully-lint-filtered LinkedIn promo copy (same resource), queues it into the next open slot as an editable pending post, and is editable from the calendar.

**Architecture:** A new `phase:'repost'` branch in the existing n8n "Lead Magnets" workflow (`XQSUuQH2e4YVwLCB`) reuses the promoter (`Claude: LinkedIn Post1`) + the scheduling-tail insert nodes, and adds a **blocking `post`-profile content-lint gate** (the gap today) with regen-on-fail. The frontend fires that webhook from a "Repost" button in `LeadMagnetEditor`; the inserted `scheduled_posts` row carries a new `is_repost` flag so the calendar opens it in `ScheduledPostEditor` (direct text edit) instead of the full LM editor.

**Tech Stack:** React + TypeScript + Tailwind (personal-site, vitest), Supabase (Postgres + edge fns + `content-lint` edge fn), n8n (REST API, `n8n-execs` skill), Railway Claude proxy.

## Global Constraints

- **Repo is `~/Desktop/personal-site`, currently on a dirty `main`.** Do all work in an **isolated git worktree** (superpowers:using-git-worktrees) and push via refspec — never commit onto the live dirty `main` (personal-site-concurrent-git-hazard).
- **n8n `XQSUuQH2e4YVwLCB` is ACTIVE (144 nodes) and PUT-publishes-live.** Re-pull the live workflow immediately before editing; concurrency-guard the PUT on the live `versionId`/`updatedAt`; touch only the new nodes + the `Status?1` switch connections; re-verify after any `activate`.
- **Copy must pass all filters:** the repost branch must route generated copy through `content-lint` (edge fn `content-lint`, profile `post`) as a **blocking** gate with regen-on-fail (max 2), plus the existing `Claude: LM QA` + `AI-Slop Gate`. Fail-open if the lint endpoint is unreachable (`lint_skipped`), fail-closed if the proxy dies on a failing draft (house pattern).
- **Scope:** lead magnets only. No regular-post repost, no bulk, no recurring cadence.
- **The repost must INSERT a new `scheduled_posts` row** (`status:'pending'`, `clickup_task_id` = the `lm_drafts_v2` id, `is_repost:true`). Never reuse/clobber the original `posted` row. Do **not** call the existing `lm-schedule` edge fn for this (it PATCHes the latest row).
- **Untouched on repost:** `resource_html`, `cover_url`, `og_url`, `slug`, the live page.
- **Supabase project ref:** `bjbvqvzbzczjbatgmccb`.
- **Motivating record (for the E2E dry run):** `lm_drafts_v2.id = cafcec96-5d15-4fd4-a4d8-ef6ed83997bc` ("The Agency Operating System for Claude Code"), last posted `scheduled_posts.id = 0124728c-171d-46fb-b503-0882cb136994` at 2026-06-24 13:00.

```ts
// lib/repostRecency.ts
/** Whole days from lastPostedAt to now (floored). null if never posted or unparseable. */
export function daysSinceLastPosted(lastPostedAtIso: string | null, nowIso: string): number | null {
  if (!lastPostedAtIso) return null;
  const then = new Date(lastPostedAtIso).getTime();
  const now = new Date(nowIso).getTime();
  if (Number.isNaN(then) || Number.isNaN(now)) return null;
  return Math.floor((now - then) / 86_400_000);
}
```

```ts
// lib/studioActions.ts — add in the "=== Lead Magnets v2 ===" block, after scheduleLM
const LM_REPOST_WEBHOOK =
  import.meta.env.VITE_LM_GEN_WEBHOOK || 'https://n8n.ivanmanfredi.com/webhook/lm-gen-v2';
const LM_REPOST_SECRET = import.meta.env.VITE_LM_REPOST_SECRET || 'pn-1ee9c4f2a7';

/**
 * Repost an already-published lead magnet: n8n regenerates fresh, lint-filtered
 * promo copy for the SAME resource and inserts a new pending scheduled_posts row
 * (is_repost=true) at the next slot. Does not touch the resource or the original post.
 */
export async function repostLeadMagnet(draft_id: string): Promise<{ ok: boolean }> {
  const res = await fetch(LM_REPOST_WEBHOOK, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ draft_id, phase: 'repost', secret: LM_REPOST_SECRET }),
  });
  if (!res.ok) throw new Error(`repost failed: ${res.status} ${(await res.text()).slice(0, 200)}`);
  return res.json().catch(() => ({ ok: true }));
}
```

```ts
// resolveLmChipTarget.ts
import type { CalendarItem } from '../../dashboard/PostCalendarView';

export type LmChipTarget =
  | { target: 'queue'; id: string }
  | { target: 'lm-editor'; id: string }
  | { target: 'error' };

/**
 * Decide what a click on an LM calendar chip opens.
 * Repost rows that are still pending (tone 'scheduled') open the queue editor so the
 * edit lands on the exact text that posts. Everything else keeps the full LM editor.
 */
export function resolveLmChipTarget(item: CalendarItem): LmChipTarget {
  if (item.isRepost && item.tone === 'scheduled') return { target: 'queue', id: item.id };
  if (item.editId) return { target: 'lm-editor', id: item.editId };
  return { target: 'error' };
}
```
